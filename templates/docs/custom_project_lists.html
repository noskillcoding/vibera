Custom drop lists

22 Aug, 2025

The drop embed allows you to inject a list of drops into your content dynamically. This can be done in any content on your space including drops, the homepage, and footer. This feature supports various filters and settings that you can apply to customize the output.

Syntax

The base syntax for embedding drops is as follows:

{% templatetag openvariable %} posts {% templatetag closevariable %}

Optional Filters and Parameters

Additional filters and parameters can be appended to the base syntax following a separator |.

1. tag: Filters drops by specific tags.

## Drops with tag "ai"
{% templatetag openvariable %} posts|tag:ai {% templatetag closevariable %}

## Drops with tag "ai" and "machine-learning"
{% templatetag openvariable %} posts|tag:ai,machine-learning {% templatetag closevariable %}

2. limit: Limits the number of drops to display.

## Check out my last 5 drops
{% templatetag openvariable %} posts|limit:5 {% templatetag closevariable %}

3. order: Orders the drops by publication date. Drops are descending by default.

## Ordered ascending
{% templatetag openvariable %} posts|order:asc {% templatetag closevariable %}

## Or descending 
{% templatetag openvariable %} posts|order:desc {% templatetag closevariable %}

4. description: Controls the visibility of the meta_description of the drop.

## Show excerpt
{% templatetag openvariable %} posts|description:True {% templatetag closevariable %}

Note: Additional CSS work will be required to format the drop list with descriptions correctly. The embedded drops have a classname of embed.

5. content: Controls the visibility of the full content of the drop.

## Show full drop content
{% templatetag openvariable %} posts|limit:3|content:True {% templatetag closevariable %}

Note: Content can only be active on pages and the homepage (not drops). Additional CSS work will be required to format the drop list with content correctly. The embedded drops have a classname of embed.

6. image: Controls the visibility of the meta_image of the drop.

## Show image
{% templatetag openvariable %} posts|limit:3|image:True {% templatetag closevariable %}

Note: Additional CSS work will be required to format the drop list with images correctly. The embedded drops have a classname of embed.

Using description, image, or content is only recommended for people who are comfortable with using CSS. The following CSS will do 90% of the work in styling the content, the rest is up to you:

ul.embedded.space-posts li {
    display: flex;
    flex-flow: row wrap;
}

/* Or using legacy class names: */
ul.embedded.blog-posts li {
    display: flex;
    flex-flow: row wrap;
}

Patterns can be combined in any order:

{% templatetag openvariable %} posts|tag:"ai"|limit:5|order:asc|description:True {% templatetag closevariable %}

Spaces are optional around the filter delimiters. The following examples are equivalent:

{% templatetag openvariable %} posts|limit:5 {% templatetag closevariable %}
{% templatetag openvariable %} posts | limit:5 {% templatetag closevariable %}
