Anatomy of the home page

22 Aug, 2025

All space information needs to be at the top of the content above the line separator, with each item on a separate line.

If the content (everything below the line separator) is left empty it will display the drops list.

title: Random space
favicon: üêº
meta_description: Vibe-code, publish, vibe-code again.
meta_image: https://example.com/image.jpeg
___

Hello, this is my solid space.

title

The title will show at the top of the space and in all space references.

favicon

optional (default: üêº)

Add a url to an image, or an emoji as a favicon.

meta_description

optional (default: first 160 characters of content)

This is the description of the space that shows up in search engines or when links are shared in messenger apps or social media. If no meta description is set, it will default to the first 160 characters of the content of the homepage.

meta_image

optional

This is a link to an image that will display when the link to this space is shared on social media or in messenger apps.

content

Content is all the text below the line separator. This is written in Markdown and will be parsed to HTML when the page is rendered. Content can also be written in pure HTML.

Filtered lists of drops (eg: Latest drops) can be inserted using filtered post list methods.

Space attributes can also be referenced in content by using the following parameters:

{% templatetag openvariable %} blog_title {% templatetag closevariable %}
{% templatetag openvariable %} blog_description {% templatetag closevariable %} 
{% templatetag openvariable %} blog_link {% templatetag closevariable %}
{% templatetag openvariable %} blog_created_date {% templatetag closevariable %}
{% templatetag openvariable %} blog_last_modified {% templatetag closevariable %} ago
{% templatetag openvariable %} blog_last_posted {% templatetag closevariable %} ago

A tag list with links to filters can be added to content using:

{% templatetag openvariable %} tags {% templatetag closevariable %}

Additionally, if you have an email signup form enabled, you can embed it using:

{% templatetag openvariable %} email-signup {% templatetag closevariable %}
